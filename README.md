## 🛡️ Sicherheitsrichtlinien

### 👤 Persönliche Sicherheit

- 🔒 Verschlüsselte Kommunikation verwenden (Signal/ProtonMail)
- 👥 Forschungskonten von persönlichen Konten trennen
- 🌐 VPN/Tor für sensible Forschung verwenden
- 💾 Backups an mehreren Orten aufbewahren

### ⚡ Operationale Sicherheit

- 🎭 Keine echten Namen in der Dokumentation verwenden
- ✅ Quellen überprüfen, bevor geteilt wird
- ⚖️ Rechtliche Implikationen in Ihrer Rechtsordnung verstehen
- 🚨 Notfallprotokolle bereit haben

### 💻 Digitale Sicherheit

- 🔄 Software aktuell halten
- 🔐 Überall 2FA verwenden
- 🔒 Sensible Daten verschlüsseln
- 🔍 Regelmäßige Sicherheits-Audits durchführen

## 🤝 Mitwirken

Nützliches gefunden? Korrekturen?

### 🚀 Schnellstart
1. **Fork** Sie dieses Repository
2. **Erstellen** Sie einen Feature-Branch (`git checkout -b feature/amazing-research`)
3. **Fügen** Sie Ihre Ergebnisse mit ordnungsgemäßer Dokumentation hinzu
4. **Testen** Sie Ihre Änderungen
5. **Reichen** Sie einen Pull Request mit detaillierter Beschreibung ein
6. **Befolgen** Sie die Mitwirkungsvorlage in [CONTRIBUTING.md](CONTRIBUTING.md)

### ✨ Wonach wir suchen

- ✅ Verifizierte Tools und Ressourcen
- 📚 Gut erforschte Fälle mit Beweisen
- 🎓 Akademische Arbeiten und Konferenzbeiträge
- 🔄 Korrekturen an vorhandenen Informationen
- 👤 Neue Ermittlerprofile mit GitHub-Repositories

### 🚫 Was wir nicht wollen

- ❌ Unverifizierte Verschwörungstheorien
- 👊 Persönliche Angriffe auf Einzelpersonen
- 🚫 Anleitungen für illegale Aktivitäten
- 💰 Kommerzieller Spam
- 🏛️ Politische Propaganda
