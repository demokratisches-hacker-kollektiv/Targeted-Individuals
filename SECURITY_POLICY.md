## 🚨 Meldung von Sicherheitslücken

Wir nehmen Sicherheit ernst. Wenn Sie eine Sicherheitslücke entdecken, melden Sie diese bitte verantwortungsvoll.

### 📧 Wie man meldet

**Primärer Kontakt:** Verwenden Sie verschlüsselte E-Mail für Sicherheitsangelegenheiten
- **E-Mail:** Erstellen Sie ein Issue mit „security“-Label für verschlüsselte Kontaktanweisungen
- **PGP-Schlüssel:** Auf Anfrage über Issue verfügbar

### 🔒 Was enthalten sein soll

1. **Art der Schwachstelle**
   - Informationsoffenlegung
   - Authentifizierungsumgehung
   - Code-Ausführung
   - Cross-Site-Scripting
   - Andere (spezifizieren)

2. **Auswirkungs-Bewertung**
   - Kritisch - Systemkompromittierung, Datenpanne
   - 🟠 Hoch - Signifikante Datenoffenlegung
   - 🟡 Mittel - Begrenzte Auswirkungen, teilweiser Zugriff
   - 🟢 Niedrig - Minimale sicherheitsrelevante Auswirkungen

3. **Reproduktionsschritte**
   - Detaillierte Schritt-für-Schritt-Anweisungen
   - Betroffene Versionen
   - Umgebungsdetails
   - Proof of Concept (falls verfügbar)

### ⏰ Reaktionszeitplan

- **🕐 24 Stunden:** Erste Bestätigung
- **🗓️ 7 Tage:** Detaillierte Bewertung und Zeitplan
- **📅 30 Tage:** Offenlegung (falls zutreffend)

## 🛡️ Sicherheits-Best Practices

### 🔍 Repository-Sicherheit
- ✅ Alle Abhängigkeiten werden regelmäßig gescannt
- ✅ Code wird auf Sicherheitsprobleme überprüft
- ✅ Keine fest codierten Anmeldeinformationen im Repository
- ✅ Sichere Kommunikationskanäle für sensible Angelegenheiten

### 👤 Benutzer-Sicherheitsrichtlinien
- 🔒 Verschlüsselte Kommunikation für sensible Angelegenheiten verwenden
- 🌐 Repository-Echtheit verifizieren (Signaturen prüfen)
- 🚫 Keine persönlichen Informationen in Issues teilen
- 🔐 2FA auf GitHub-Konten aktivieren

### 🏢 Operationale Sicherheit
- 🎭 Anonymität für sensible Forschung wahren
- 📸 Beweise sicher dokumentieren
- 🔍 Quellen vor dem Teilen verifizieren
- 🚨 Notfallreaktionsverfahren haben

## 🚫 Verbotene Inhalte

Folgende Inhalte sind strengstens verboten und führen zur sofortigen Entfernung:

- ❌ Anleitungen für illegale Aktivitäten
- 🚫 Bösartiger Code oder Tools
- 👊 Persönliche Angriffe oder Belästigung
- 💰 Kommerzielle Ausbeutung
- 🎭 Identitätsdiebstahl von Ermittlern

## 🔐 Verschlüsselungsrichtlinien

### 📧 E-Mail-Kommunikation
- **Bevorzugt:** PGP/GPG-verschlüsselte E-Mails
- **Alternative:** ProtonMail oder Signal
- **Minimum:** TLS-verschlüsselte E-Mail

### 📁 Dateifreigabe
- **Empfohlen:** Verschlüsselte Container (7z, AES-256)
- **Schlüssel:** Verschlüsselungsschlüssel getrennt teilen
- **Verifizierung:** Dateiintegrität immer überprüfen

## 🚨 Vorfallreaktion

### 📊 Vorfall-Klassifizierung
- **Level 1:** Informationsoffenlegung (öffentliche Quellen)
- **Level 2:** Private Datenoffenlegung
- **Level 3:** Systemkompromittierung
- **Level 4:** Umfassende Auswirkungen

### 🔄 Reaktionsverfahren
1. **🚨 Erkennung** - Sofortige Identifizierung
2. **📝 Bewertung** - Auswirkungsanalyse
3. **🔒 Eindämmung** - Exposition begrenzen
4. **🔧 Behebung** - Reparieren und Patchen
5. **📚 Dokumentation** - Gelernte Lektionen
